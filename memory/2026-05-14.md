## Technical Lessons

### Oracle Usage
- Command: `npx -y @steipete/oracle --engine browser --browser-manual-login --generate-image <output_path> --file <ref1> --file <ref2> -p "<prompt>"`
- Must be serial, one at a time
- Can take 10+ minutes per image; don't set short timeouts
- `--browser-manual-login` required for browser mode
- Reference images passed via `--file`; prompt passed via `-p`
- Output saved to path specified in `--generate-image`

### Codex Exec Fallback (Allowed)
- User approved: if Oracle fails, use `codex` exec to enter a temp folder and execute image generation tasks
- Codex exec = local PTY Codex session that can run image generation commands in a workspace directory
- Use only as fallback when Oracle fails, not as primary

### Failed Approaches (Don't Repeat)
- `image_generate` tool: blocked with "resolves to private/internal/special-use IP address" for local files
- ACP Codex threads: unreliable, often only produce text planning without actual file output
- Codex exec local: produced wrong product images (e.g. wooden box instead of cooler tote)
- Subagent parallel Oracle: times out because subagents have default timeout; Oracle needs long runtime

### Working Approaches
- Oracle browser mode: reliable for image generation, serial only
- `lark-cli base +record-upload-attachment`: works for most uploads
- Raw `drive/v1/medias/upload_all` + PUT: fallback when lark-cli upload fails
