## 长任务自动停止排查

- Context: 用户问为什么每次运行 1-2 小时左右就自动停止、不继续运行。
- Evidence: OpenClaw 配置里 `tools.exec.timeoutSec=1800`，单个 exec 工具最多约 30 分钟；Oracle 浏览器生图自身常提示最长可运行约 1 小时。最近 ASI 任务轨迹显示北京时间约 00:37 继续、01:34 `session.ended`，属于当前 turn 完成后结束，不是常驻后台 worker 自动续跑。
- Lesson: 普通 Feishu 对话 turn 不能被当作“会一直后台跑”的任务。跨小时 ASI 批量任务要显式拆分为可恢复队列，并用 cron、heartbeat、后台 session 或分批交付来续跑；若没有这些机制，完成一次回复或遇到工具阻塞/等待确认后就会停止。

## Turn 结束不是压缩直接杀掉

- Context: 用户进一步怀疑“一个 turn 为什么这么短；是不是 Codex harness 压缩后就结束 turn”。
- Evidence: 轨迹里每轮顺序是 `context.compiled -> prompt.submitted -> tool.call/tool.result... -> model.completed -> session.ended`；最近 `model.completed` 记录显示 `timedOut=false`、`yieldDetected=false`、`aborted=false`，随后 `session.ended status=success`。OpenClaw 配置为 `agents.defaults.compaction.mode=safeguard`、`plugins.slots.contextEngine=lossless-claw`，但未发现“压缩后结束 turn”的配置或日志证据。`openclaw status` 显示当前 direct session 约 `199k/272k`，有上下文压力但未满。
- Lesson: 当前证据更支持“模型认为已完成/主动 final 后 harness 正常收尾”，而不是“compaction 自动杀 turn”。长任务中应避免过早 final/停问，或用后台/队列机制让后续步骤明确存在。

## Turn 结束表述更正：不要把 completed 等同 stop

- Context: 用户追问“模型为什么会发出 stop？真的是模型发 stop 的原因吗？”
- Evidence: 重新检查 raw trajectory 后，只能看到 `model.completed`、`timedOut=false`、`aborted=false`、`yieldDetected=false`、`promptError=null`，没有看到原始 API 的 `finish_reason: stop` 字段。
- Lesson: 以后排查 turn 结束时，不能把 OpenClaw 轨迹里的 `model.completed` 直接说成“模型发了 stop”。更准确说法是：模型响应正常完成且没有继续产生 tool call，runtime 因此结束 turn。若要证明 API-level stop，需要找到原始响应里的 finish/stop reason。

## Finalize Guard 设计偏好

- Context: 用户想研究 OpenClaw 是否能在一个 turn 准备结束时再让 LLM 自检：如果确实该结束则输出 `SESSION_OK`，否则继续任务；并希望清理该检查提示，避免污染上下文。
- Evidence: 已查到 OpenClaw plugin typed hook 中存在 `before_agent_finalize`，返回值支持 `action: "revise"` 与 `retry.instruction`，适合在自然 final 被接受前请求模型再跑一轮；这比“结束后再开一轮”更干净。hook 事件可提供 `sessionKey/sessionId/turnId/cwd/transcriptPath/lastAssistantMessage/messages/stopHookActive` 等。
- User preference: 用户坚决不同意写 ASI 专用、progress.json 专用或特定任务规则；核心思想应保持由 LLM 自己判断是否真的该结束。若存在 `update_plan` 且有未完成任务，可作为提示给 LLM，但不要把特定业务 pending 文件作为硬判断。
- User preference: 不要把“不要 ASI 专用规则”等负规则写进正式文件，也不要在插件代码里写 ASI、progress.json、特定任务相关注释或判断。正式代码应保持通用 final self-check guard。
- Proposed implementation (not yet executed): 新增 `~/.openclaw/extensions/session-finalize-guard/`，包含 `package.json` 和 `index.js`；注册 `before_agent_finalize`，若最后输出精确为 `SESSION_OK` 则允许结束，否则返回 `action:"revise"`，指令让 LLM 自查是否确实应结束；可选从最近 `update_plan` 提取未完成项作为提示。修改 `~/.openclaw/openclaw.json` 仅用于启用插件。
- Testing plan discussed: 先用 `openclaw hooks list/check` 验证加载；用简单问答确认会触发自检并能以 `SESSION_OK` 正常结束；再用带 `update_plan` 的未完成任务测试自检提示；观察 `retry.maxAttempts` 是否被 runtime 尊重，不能假设存在 `maxRevisionsPerTurn` 配置。
