# ASI 商品主图持续生成

从 LogoPress ERP「商品表」持续筛选：商品主图为空、参考图片不为空的商品。筛选与选品只由主调度者执行：每次派发前，主调度者先选定一个具体商品，记录其商品编号、record_id、当前记录版本和参考图附件 token，并建立该商品的唯一占用记录。然后派遣 subagent，并在任务正文中明确写出“只处理商品 LPxxxx、record_id=xxxx”；子任务不得自行到表中提取、筛选、替换或猜测要处理的商品，也不得处理任何其他 record。等待该指定商品的 subagent 返回结果；只有占用记录被明确释放、且回读确认该商品没有并发变更后，才可为下一个已明确指定的商品重新派发。subagent 完成后，主调度者再从 LogoPress ERP「商品表」筛选并明确指定下一个商品，循环直到所有符合条件的商品都完成 ASI 生图。每个 subagent 只负责一个已指定商品；同一商品在任何时刻只允许一个执行流（包括遗留 Oracle 会话和重试）存在。可同时运行两个不同商品的 subagent。生成的图片发给我；每次心跳检查 subagent 运行状态，主动解决问题、恢复或重试，但重试前必须确认旧会话已终止并重新取得该商品唯一占用。每次 subagent 完成时要像念经一样完整复述本段任务内容。

运行约束：每次心跳先检查现有 subagent 和全部未释放占用记录；空闲槽位最多补派至 2 个，且只能补派不同且已明确指定的商品。每个子任务必须读取并遵守 `skills/asi-product-image-generator/SKILL.md`，仅处理任务中给出的商品编号与 record_id；成功后将生成图片发给用户并回写商品表。通过 Oracle 启动 ChatGPT 生图时，使用 Medium 思考强度；如按模型档位选择，则使用等效 Medium 模型。任务完成时清空本文件。
